ThinkSQL™ RDBMS

 

Release Notes

13th September 2004

 

Version 01.04.00

Fixes

Fixed WITH RETURN cursors.

Fixed CAST from timestamp to char.

Fix variable reference scope problem within cursors.

JDBC database metadata getTypeInfo returns DATA_TYPE.

CREATE INDEX (undocumented) is no longer ignored.

 

Enhancements

Added JDBC SetObject method.

Added DEBUG PRINT statement to ease routine debugging.

 

Version 01.03.00

Fixes

Fixed bug with routines partially executing IF/ELSE/CASE compound statements.

ODBC SQLDriverConnect returns output string.

JDBC getObject returns correct day for timestamps.

 

Enhancements

Added Python DB-API driver (see ThinkSQLPython-1.03.zip).

Allow routines to be up to 2M in size (for new catalogs).

 

Version 01.02.00

Fixes

Improved reporting of function result types during statement preparation.

dbExpress now passes transaction isolation level to server.

JDBC fixes include:

  • parameter fix
  • speed increase for getImported(Exported)Keys
  • fix for missing column_def in getColumns
  • throw exception for unimplemented options
  • fix for getBestRowIndentifier/getVersionColumns/getProcedureColumns

 

Version 01.01.00

Fixes

Dropped and recreated sequences in a single server session now clear any remaining cached values.

dbExpress, ODBC and JDBC driver metadata for numeric column precision is now reported correctly.

dbExpress TSQLconnection.StartTransaction now temporarily disables autocommit until Commit or Rollback.

JDBC fixes include:

  • exception reporting improved
  • autocommit is now on by default
  • fix for empty string results (was returning null character)
  • getColumns returns java.sql.Types (rather than ODBC types)
  • information_schema views are now reported via getTables as System Tables (previously they were only reported if connected as information_schema)
  • metadata fix for getColumn nullable/is_nullable columns

 

Version 01.00.00

Initial release.
